rollback_chat

This function is used to send a string message to other players.

You specify a string to send, with a maximum size of 128 bytes.

When the second argument is not specified, the message is sent to all players, including the sender.

You can specify a player ID in the second argument which is the player who will receive the message.

How to Use?

This function is meant to be used "locally", and so would result from local input, not Rollback inputs.

See the usage example at the bottom. Listen for received messages in the Rollback Event event. 

 

Syntax:

rollback_chat(message, [to]);

Argument Type Description
message String The message to send, as a string.
to Real OPTIONAL The ID of the player to which the message will be sent. Not specifying this argument, or specifying -1, sends the message to all players.

 

Returns:

N/A

 

Example:

if (keyboard_check_pressed(vk_enter))
{
    rollback_chat(keyboard_string);
    keyboard_string = "";
}

If the enter key is pressed locally, the keyboard_string value is sent to all players. That string is then cleared so a new message can be typed.

This will trigger the Rollback Event event when the message is received by all clients (including the sender).

Rollback Event

if (rollback_event_id == rollback_chat_message)
{
    // Get received information
    var _message = rollback_event_param.message;
    var _from = rollback_event_param.from;

    // Find sender's name
    var _name = rollback_get_info(_from).player_name;

    // Print to output log
    show_debug_message(_name + " says: " + _message);
}
