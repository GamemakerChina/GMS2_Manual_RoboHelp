rollback_define_mock_input

This function can be used during Sync Test to assign temporary "mock" input to the specified player. This overrides the controls defined for the player and any random input from Sync Test.

For information on its use, see Mock Input.

 

Syntax:

rollback_define_mock_input(player_id, input_struct);

Argument Type Description
player_id Real The ID of the player to assign mock input to.
input_struct Struct A struct containing input names as variables, and their assigned inputs as values.

 

Returns:

N/A

 

Example:

rollback_define_input({
    fire: mb_left,
    interact: vk_space,
    left: ord("A"),
    right: ord("D")
});

rollback_define_mock_input(1, {
    fire: vk_control,
    interact: vk_shift,
    left: ord("J"),
    right: ord("L")
});

The code above first defines the inputs for players. It then defines mock input for player 1, keeping the same input names but assigning different inputs. This way two people could play the game locally using the keyboard.
